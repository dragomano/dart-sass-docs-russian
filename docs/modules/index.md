> Sass предоставляет множество встроенных модулей, содержащих полезные функции (и иногда миксины). Эти модули можно загружать с помощью правила [`@use`](../at-rules/use) как любую пользовательскую таблицу стилей, а их функции можно вызывать [как любые другие элементы модуля](../at-rules/use#loading-members). Все URL встроенных модулей начинаются с `sass:`, что указывает на их принадлежность к Sass.

!!! note "Примечание"

    До введения системы модулей все функции Sass были глобально доступны всегда. Многие функции по-прежнему имеют глобальные псевдонимы (они перечислены в их документации). Команда Sass не рекомендует их использовать и со временем их удалит, но пока они остаются доступными для совместимости со старыми версиями Sass и LibSass (которая ещё не поддерживает систему модулей).

    [Некоторые функции](#global-functions) доступны *только* глобально даже в новой системе модулей, поскольку они добавляют дополнительное поведение поверх встроенных функций CSS.

<div class="grid" markdown>

=== "SCSS"

    ```scss
    @use "sass:color";

    .button {
      $primary-color: #6b717f;
      color: $primary-color;
      border: 1px solid color.scale($primary-color, $lightness: 20%);
    }
    ```

=== "SASS"

    ```sass
    @use "sass:color"

    .button
      $primary-color: #6b717f
      color: $primary-color
      border: 1px solid color.scale($primary-color, $lightness: 20%)
    ```

```css title="CSS"
.button {
  color: #6b717f;
  border: 1px solid rgb(135.1641025641, 140.8256410256, 154.0358974359);
}
```

</div>

Sass предоставляет следующие встроенные модули:

* Модуль [`sass:math`](../modules/math) содержит функции для работы с [числами](../values/numbers).

* Модуль [`sass:string`](../modules/string) упрощает объединение, поиск и разделение [строк](../values/strings).

* Модуль [`sass:color`](../modules/color) создает новые [цвета](../values/colors) на основе существующих, облегчая создание цветовых тем.

* Модуль [`sass:list`](../modules/list) позволяет получать и изменять значения в [списках](../values/lists).

* Модуль [`sass:map`](../modules/map) обеспечивает поиск значения по ключу в [карте](../values/maps) и многое другое.

* Модуль [`sass:selector`](../modules/selector) предоставляет доступ к мощному селекторному движку Sass.

* Модуль [`sass:meta`](../modules/meta) раскрывает внутренние механизмы работы Sass.

## Глобальные функции {#global-functions}

!!! tip "Совет"

    Вы можете передавать [специальные функции](../syntax/special-functions), такие как `calc()` или `var()`, вместо любого аргумента глобальному конструктору цвета. Вы даже можете использовать `var()` вместо нескольких аргументов, поскольку он может быть заменен несколькими значениями! При вызове цветовой функции таким образом она возвращает строку без кавычек с той же сигнатурой, с которой была вызвана.

    === "SCSS"

        ```scss
        @debug rgb(0 51 102 / var(--opacity)); // rgb(0 51 102 / var(--opacity))
        @debug color(display-p3 var(--peach)); // color(display-p3 var(--peach))
        ```

    === "SASS"

        ```sass
        @debug rgb(0 51 102 / var(--opacity)) // rgb(0 51 102 / var(--opacity))
        @debug color(display-p3 var(--peach)) // color(display-p3 var(--peach))
        ```

### color

```scss
color($space $channel1 $channel2 $channel3)
color($space $channel1 $channel2 $channel3 / $alpha) //=> color
```

Возвращает цвет в указанном цветовом пространстве с заданными значениями каналов.

Поддерживает цветовые пространства `srgb`, `srgb-linear`, `display-p3`, `a98-rgb`, `prophoto-rgb`, `rec2020`, `xyz` и `xyz-d50`, а также `xyz-d65`, являющийся синонимом для `xyz`. Для всех пространств каналы представляют собой числа от 0 до 1 (включительно) или проценты от `0%` до `100%` (включительно).

Если какой-либо канал цвета выходит за пределы диапазона от 0 до 1, это представляет цвет за пределами стандартной гаммы для данного цветового пространства.

=== "SCSS"

    ```scss
    @debug color(srgb 0.1 0.6 1); // color(srgb 0.1 0.6 1)
    @debug color(xyz 30% 0% 90% / 50%); // color(xyz 0.3 0 0.9 / 50%)
    ```

=== "SASS"

    ```sass
    @debug color(srgb 0.1 0.6 1) // color(srgb 0.1 0.6 1)
    @debug color(xyz 30% 0% 90% / 50%) // color(xyz 0.3 0 0.9 / 50%)
    ```

### hsl

```scss
hsl($hue $saturation $lightness)
hsl($hue $saturation $lightness / $alpha)
hsl($hue, $saturation, $lightness, $alpha: 1)
hsla($hue $saturation $lightness)
hsla($hue $saturation $lightness / $alpha)
hsla($hue, $saturation, $lightness, $alpha: 1) //=> color
```

Возвращает цвет с заданными [оттенком (hue), насыщенностью (saturation), яркостью (lightness)](https://en.wikipedia.org/wiki/HSL_and_HSV) и указанным альфа-каналом.

Оттенок — число от `0deg` до `360deg` (включительно), может быть без единиц измерения. Насыщенность и яркость обычно — числа от `0%` до `100%` (включительно) и *не могут* быть без единиц измерения. Альфа-канал можно указать либо как число без единиц от 0 до 1 (включительно), либо как процент от `0%` до `100%` (включительно).

Оттенок вне диапазона `0deg`–`360deg` эквивалентен `$hue % 360deg`. Насыщенность меньше `0%` обрезается до `0%`. Насыщенность выше `100%` или яркость вне диапазона `0%`–`100%` допустимы и представляют цвета за пределами стандартной RGB-гаммы.

!!! note "Примечание"

    Специальные [правила разбора Sass](../operators/numeric#slash-separated-values) для значений, разделённых слешами, затрудняют передачу переменных для `$lightness` или `$alpha` при использовании сигнатуры `hsl($hue $saturation $lightness / $alpha)$. Рекомендуется использовать `hsl($hue, $saturation, $lightness, $alpha)` вместо этого.

=== "SCSS"

    ```scss
    @debug hsl(210deg 100% 20%); // #036
    @debug hsl(210deg 100% 20% / 50%); // rgba(0, 51, 102, 0.5)
    @debug hsla(34, 35%, 92%, 0.2); // rgba(241.74, 235.552, 227.46, 0.2)
    ```

=== "SASS"

    ```sass
    @debug hsl(210deg 100% 20%) // #036
    @debug hsl(210deg 100% 20% / 50%) // rgba(0, 51, 102, 0.5)
    @debug hsla(34, 35%, 92%, 0.2) // rgba(241.74, 235.552, 227.46, 0.2)
    ```

### hwb

```scss
hwb($hue $whiteness $blackness)
hwb($hue $whiteness $blackness / $alpha)
color.hwb($hue $whiteness $blackness)
color.hwb($hue $whiteness $blackness / $alpha)
color.hwb($hue, $whiteness, $blackness, $alpha: 1) //=> color
```

Возвращает цвет с заданными [оттенком (hue), белизной (whiteness), чернотой (blackness)](https://en.wikipedia.org/wiki/HWB_color_model) и указанным альфа-каналом.

Оттенок — число от `0deg` до `360deg` (включительно), может быть без единиц измерения. Белизна и чернота обычно — числа от `0%` до `100%` (включительно) и *не могут* быть без единиц измерения. Альфа-канал можно указать либо как число без единиц от 0 до 1 (включительно), либо как процент от `0%` до `100%` (включительно).

Оттенок вне диапазона `0deg`–`360deg` эквивалентен `$hue % 360deg`. Если `$whiteness + $blackness > 100%`, оба значения масштабируются так, чтобы в сумме составлять `100%`. Если `$whiteness`, `$blackness` или оба значения меньше `0%`, это представляет цвет за пределами стандартной RGB-гаммы.

!!! note "Примечание"

    Варианты `color.hwb()` устарели. Новый код Sass должен использовать глобальную функцию `hwb()` вместо этого.

=== "SCSS"

    ```scss
    @debug hwb(210deg 0% 60%); // #036
    @debug hwb(210 0% 60% / 0.5); // rgba(0, 51, 102, 0.5)
    ```

=== "SASS"

    ```sass
    @debug hwb(210deg 0% 60%) // #036
    @debug hwb(210 0% 60% / 0.5) // rgba(0, 51, 102, 0.5)
    ```

### lab

```scss
lab($lightness $a $b)
lab($lightness $a $b / $alpha) //=> color
```

Возвращает цвет с заданными каналами [яркости (lightness), a, b](https://en.wikipedia.org/wiki/CIELAB_color_space) и альфа-каналом.

Яркость — число от `0%` до `100%` (включительно), может быть без единиц измерения. Каналы a и b можно указать либо как [числа без единиц](../values/numbers#units) от -125 до 125 (включительно), либо как проценты от `-100%` до `100%` (включительно). Альфа-канал можно указать либо как число без единиц от 0 до 1 (включительно), либо как процент от `0%` до `100%` (включительно).

Яркость вне диапазона `0%`–`100%` обрезается до этого диапазона. Если каналы a или b находятся за пределами диапазона `-125`–`125`, это представляет цвет за пределами стандартной CIELAB-гаммы.

=== "SCSS"

    ```scss
    @debug lab(50% -20 30); // lab(50% -20 30)
    @debug lab(80% 0% 20% / 0.5); // lab(80% 0 25 / 0.5);
    ```

=== "SASS"

    ```sass
    @debug lab(50% -20 30) // lab(50% -20 30)
    @debug lab(80% 0% 20% / 0.5) // lab(80% 0 25 / 0.5);
    ```

### lch

```scss
lch($lightness $chroma $hue)
lch($lightness $chroma $hue / $alpha) //=> color
```

Возвращает цвет с заданными [яркостью (ligthness), цветностью (chroma), оттенком (hue)](https://en.wikipedia.org/wiki/CIELAB_color_space#Cylindrical_model) и указанным альфа-каналом.

Яркость — число от `0%` до `100%` (включительно), может быть без единиц измерения. Канал цветности можно указать либо как [число без единиц](../values/numbers#units) от 0 до 150 (включительно), либо как процент от `0%` до `100%` (включительно). Оттенок — число от `0deg` до `360deg` (включительно), может быть без единиц измерения. Альфа-канал можно указать либо как число без единиц от 0 до 1 (включительно), либо как процент от `0%` до `100%` (включительно).

Яркость вне диапазона `0%`–`100%` обрезается до этого диапазона. Цветность ниже 0 обрезается до 0, а цветность выше 150 представляет цвет за пределами стандартной CIELAB-гаммы. Оттенок вне диапазона `0deg`–`360deg` эквивалентен `$hue % 360deg`.

=== "SCSS"

    ```scss
    @debug lch(50% 10 270deg); // lch(50% 10 270deg)
    @debug lch(80% 50% 0.2turn / 0.5); // lch(80% 75 72deg / 0.5);
    ```

=== "SASS"

    ```sass
    @debug lch(50% 10 270deg) // lch(50% 10 270deg)
    @debug lch(80% 50% 0.2turn / 0.5) // lch(80% 75 72deg / 0.5);
    ```

Яркость — число от `0%` до `100%` (включительно), может быть без единиц измерения. Каналы a и b можно указать либо как [числа без единиц](../values/numbers#units) от -0.4 до 0.4 (включительно), либо как проценты от `-100%` до `100%` (включительно). Альфа-канал можно указать либо как число без единиц от 0 до 1 (включительно), либо как процент от `0%` до `100%` (включительно).

Яркость вне диапазона `0%`–`100%` обрезается до этого диапазона. Если каналы a или b находятся за пределами диапазона `-0.4`–`0.4`, это представляет цвет за пределами стандартной Oklab-гаммы.

### rgb

```scss
rgb($red $green $blue)
rgb($red $green $blue / $alpha)
rgb($red, $green, $blue, $alpha: 1)
rgb($color, $alpha)
rgba($red $green $blue)
rgba($red $green $blue / $alpha)
rgba($red, $green, $blue, $alpha: 1)
rgba($color, $alpha) //=> color
```

Если переданы `$red`, `$green`, `$blue` и опционально `$alpha`, возвращает цвет с указанными каналами красного, зеленого, синего и альфа-каналом.

Каждый канал можно указать либо как [число без единиц](../values/numbers#units) от 0 до 255 (включительно), либо как процент от `0%` до `100%` (включительно). Альфа-канал можно указать либо как число без единиц от 0 до 1 (включительно), либо как процент от `0%` до `100%` (включительно).

Если какой-либо канал цвета находится за пределами диапазона 0–255, это представляет цвет за пределами стандартной RGB-гаммы.

!!! tip "Совет"

    Специальные [правила разбора Sass](../operators/numeric#slash-separated-values) для значений, разделённых слешами, затрудняют передачу переменных для `$blue` или `$alpha` при использовании сигнатуры `rgb($red $green $blue / $alpha)`. Рекомендуется использовать `rgb($red, $green, $blue, $alpha)` вместо этого.

=== "SCSS"

    ```scss
    @debug rgb(0 51 102); // #036
    @debug rgb(95%, 92.5%, 89.5%); // #f2ece4
    @debug rgb(0 51 102 / 50%); // rgba(0, 51, 102, 0.5)
    @debug rgba(95%, 92.5%, 89.5%, 0.2); // rgba(242, 236, 228, 0.2)
    ```

=== "SASS"

    ```sass
    @debug rgb(0 51 102) // #036
    @debug rgb(95%, 92.5%, 89.5%) // #f2ece4
    @debug rgb(0 51 102 / 50%) // rgba(0, 51, 102, 0.5)
    @debug rgba(95%, 92.5%, 89.5%, 0.2) // rgba(242, 236, 228, 0.2)
    ```

---

Если переданы `$color` и `$alpha`, возвращает `$color` с указанным `$alpha` каналом вместо его оригинального альфа-канала.

=== "SCSS"

    ```scss
    @debug rgb(#f2ece4, 50%); // rgba(242, 236, 228, 0.5);
    @debug rgba(rgba(0, 51, 102, 0.5), 1); // #003366
    ```

=== "SASS"

    ```sass
    @debug rgb(#f2ece4, 50%) // rgba(242, 236, 228, 0.5)
    @debug rgba(rgba(0, 51, 102, 0.5), 1) // #003366
    ```

## Устаревшие функции {#deprecated-functions}

### if

```scss
if($condition, $if-true, $if-false)
```

Возвращает `$if-true`, если `$condition` является [истинным значением](../at-rules/control/if#truthiness-and-falsiness), и `$if-false` в противном случае.

Эта функция особенная тем, что не вычисляет аргумент, который не возвращается, поэтому безопасно вызывать её даже если неиспользуемый аргумент вызовет ошибку.

!!! note "Примечание"

    Теперь, когда CSS поддерживает собственный [синтаксис функции `if()`](../syntax/special-functions#if), Sass переходит на использование этого синтаксиса. Старая функция `if()` устарела, хотя будет поддерживаться в Dart Sass до версии 3.0.0. Подробнее см. [/d/if-function](../breaking-changes/if-function).

=== "SCSS"

    ```scss
    @debug if(true, 10px, 15px); // 10px
    @debug if(false, 10px, 15px); // 15px
    @debug if(variable-defined($var), $var, null); // null
    ```

=== "SASS"

    ```sass
    @debug if(true, 10px, 15px) // 10px
    @debug if(false, 10px, 15px) // 15px
    @debug if(variable-defined($var), $var, null) // null
    ```
