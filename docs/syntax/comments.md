> Работа комментариев Sass существенно различается между SCSS и синтаксисом с отступами. Оба синтаксиса поддерживают два типа комментариев: комментарии, определённые с помощью `/* */`, которые (обычно) компилируются в CSS, и комментарии, определённые с помощью `//`, которые не компилируются.

## В SCSS {#in-scss}

Комментарии в SCSS работают аналогично комментариям в других языках, таких как JavaScript. **Однострочные комментарии** начинаются с `//` и продолжаются до конца строки. Ничего из однострочного комментария не выводится в CSS; с точки зрения Sass, они могут вообще не существовать. Их также называют **тихими комментариями**, потому что они не производят никакого CSS.

**Многострочные комментарии** начинаются с `/*` и заканчиваются следующим `*/`. Если многострочный комментарий написан там, где разрешено [выражение](structure#statements), он компилируется в CSS-комментарий. Их также называют **громкими комментариями**, в отличие от тихих комментариев. Многострочный комментарий, который компилируется в CSS, может содержать [интерполяцию](/interpolation), которая будет вычислена до компиляции комментария.

По умолчанию многострочные комментарии будут удалены из скомпилированного CSS в [сжатом режиме](/cli/dart-sass/#style). Однако если комментарий начинается с `/*!`, он всегда будет включён в вывод CSS.

=== "SCSS"

    ```scss
    // Этот комментарий не будет включён в CSS.

    /* А этот комментарий будет, за исключением сжатого режима. */

    /* Он также может содержать интерполяцию:
    * 1 + 1 = #{1 + 1} */

    /*! Этот комментарий будет включён даже в сжатом режиме. */

    p /* Многострочные комментарии можно писать в любом месте,
      * где разрешены пробелы. */ .sans {
      font: Helvetica, // Так же, как и однострочные комментарии.
            sans-serif;
    }
    ```

=== "CSS"

    ```css
    /* А этот комментарий будет, за исключением сжатого режима. */
    /* Он также может содержать интерполяцию:
    * 1 + 1 = 2 */
    /*! Этот комментарий будет включён даже в сжатом режиме. */
    p .sans {
      font: Helvetica, sans-serif;
    }
    ```

## В Sass {#in-sass}

Комментарии в синтаксисе с отступами работают немного иначе: они основаны на отступах, как и весь остальной синтаксис. Как и в SCSS, тихие комментарии, написанные с `//`, никогда не выводятся в CSS, но в отличие от SCSS всё, что имеет отступ под открывающим `//`, также закомментировано.

Комментарии синтаксиса с отступами, которые начинаются с `/*`, работают с отступами таким же образом, за исключением того, что они компилируются в CSS. Поскольку объём комментария основан на отступах, закрывающий `*/` необязателен. Также, как и в SCSS, комментарии `/*` могут содержать [интерполяцию](/interpolation) и могут начинаться с `/*!`, чтобы избежать удаления в сжатом режиме.

Комментарии также могут использоваться внутри [выражений](structure#expressions) в синтаксисе с отступами. В этом случае они имеют точно такой же синтаксис, как и в SCSS.

=== "Sass"

    ```sass
    // Этот комментарий не будет включён в CSS.
      Это также закомментировано.

    /* А этот комментарий будет, за исключением сжатого режима.

    /* Он также может содержать интерполяцию:
      1 + 1 = #{1 + 1}

    /*! Этот комментарий будет включён даже в сжатом режиме.

    p .sans
      font: Helvetica, /* Встроенные комментарии должны быть закрыты. */ sans-serif
    ```

=== "CSS"

    ```css
    /* А этот комментарий будет, за исключением сжатого режима. */
    /* Он также может содержать интерполяцию:
     * 1 + 1 = 2 */
    /*! Этот комментарий будет включён даже в сжатом режиме. */
    p .sans {
      font: Helvetica, sans-serif;
    }
    ```

## Документирующие комментарии {#documentation-comments}

При написании библиотек стилей с использованием Sass вы можете использовать комментарии для документирования [миксинов](/at-rules/mixin), [функций](/at-rules/function), [переменных](/variables) и [селекторов-заполнителей](/style-rules/placeholder-selectors), которые предоставляет ваша библиотека, а также самой библиотеки. Эти комментарии читаются инструментом [SassDoc](https://sassdoc.com), который использует их для создания красивой документации. Посмотрите на документацию [движка сеток Susy](https://oddbird.net/susy/docs/index.html), чтобы увидеть его в действии!

Документирующие комментарии — это тихие комментарии, написанные с тремя косыми чертами (`///`) непосредственно над тем, что вы документируете. SassDoc анализирует текст в комментариях как [Markdown](https://www.markdownguide.org/getting-started) и поддерживает множество полезных [аннотаций](https://sassdoc.com/annotations/) для детального описания.

=== "SCSS"

    ```scss
    /// Вычисляет степень числа.
    ///
    /// @param {number} $base
    ///   Число, которое нужно умножить само на себя.
    /// @param {integer (unitless)} $exponent
    ///   Количество `$base`, которые нужно перемножить.
    /// @return {number} `$base` в степени `$exponent`.
    @function pow($base, $exponent) {
        $result: 1;
        @for $_ from 1 through $exponent {
          $result: $result * $base;
        }
        @return $result;
    }
    ```

=== "Sass"

    ```sass
    /// Вычисляет степень числа.
    ///
    /// @param {number} $base
    ///   Число, которое нужно умножить само на себя.
    /// @param {integer (unitless)} $exponent
    ///   Количество `$base`, которые нужно перемножить.
    /// @return {number} `$base` в степени `$exponent`.
    @function pow($base, $exponent)
        $result: 1
        @for $_ from 1 through $exponent
          $result: $result * $base

        @return $result
    ```
