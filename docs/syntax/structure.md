> Так же, как и CSS, большинство таблиц стилей Sass в основном состоят из правил стилей, содержащих декларации свойств. Но таблицы стилей Sass имеют гораздо больше возможностей, которые могут существовать наряду с ними.

## Выражения {#statements}

Таблица стилей Sass состоит из серии _выражений_, которые выполняются по порядку для построения итогового CSS. Некоторые выражения могут иметь _блоки_, определённые с помощью `{` и `}`, которые содержат другие выражения. Например, правило стиля — это выражение с блоком. Этот блок содержит другие выражения, такие как декларации свойств.

В SCSS выражения разделяются точками с запятой (которые необязательны, если выражение использует блок). В синтаксисе с отступами они просто разделяются переносами строк. Хотя вы можете использовать точку с запятой в конце выражения в синтаксисе с отступами, перенос строки всё равно требуется.

### Универсальные выражения {#universal-statements}

Эти типы выражений могут использоваться в любом месте таблицы стилей Sass:

- [Объявления переменных](/variables), например `$var: value`.
- [@-правила управления потоком](/at-rules/control), например `@if` и `@each`.
- Правила [`@error`](/at-rules/error), [`@warn`](/at-rules/warn) и [`@debug`](/at-rules/debug).

### CSS-выражения {#css-statements}

Эти выражения создают CSS. Они могут использоваться где угодно, за исключением внутри `@function`:

- [Правила стилей](/style-rules), например `h1 { /* ... */ }`.
- [CSS @-правила](/at-rules/css), например `@media` и `@font-face`.
- [Использование миксинов](/at-rules/mixin) с помощью `@include`.
- [Правило `@at-root`](/at-rules/at-root).

### Выражения верхнего уровня {#top-level-statements}

Эти выражения могут использоваться только на верхнем уровне таблицы стилей или вложенными в CSS-выражение на верхнем уровне:

- [Загрузка модулей](/at-rules/use) с помощью `@use`.
- [Импорты](/at-rules/import) с помощью `@import`.
- [Определения миксинов](/at-rules/mixin) с помощью `@mixin`.
- [Определения функций](/at-rules/function) с помощью `@function`.

### Другие выражения {#other-statements}

- [Декларации свойств](/style-rules/declarations), такие как `width: 100px`, могут использоваться только внутри правил стилей и некоторых @-правил CSS.
- [Правило `@extend`](/at-rules/extend) может использоваться только внутри правил стилей.

## Выражения {#expressions}

_Выражение_ — это всё, что находится в правой части декларации свойства или переменной. Каждое выражение создаёт _[значение](/values)_. Любое валидное значение CSS-свойства также является выражением Sass, но выражения Sass гораздо мощнее, чем простые значения CSS. Они передаются как аргументы [миксинам](/at-rules/mixin) и [функциям](/at-rules/function), используются для управления потоком с [правилом `@if`](/at-rules/control/if) и обрабатываются с помощью [арифметики](/operators/numeric). Мы называем синтаксис выражений Sass _SassScript_.

### Литералы {#literals}

Простейшие выражения просто представляют статические значения:

- [Числа](/values/numbers), которые могут иметь или не иметь единицы измерения, например `12` или `100px`.
- [Строки](/values/strings), которые могут быть заключены или не заключены в кавычки, например `"Helvetica Neue"` или `bold`.
- [Цвета](/values/colors), на которые можно ссылаться по их шестнадцатеричному представлению или по имени, например `#c6538c` или `blue`.
- [Логические](/values/booleans) литералы `true` или `false`.
- Одиночное значение [`null`](/values/null).
- [Списки значений](/values/lists), которые могут быть разделены пробелами или запятыми и могут быть заключены в квадратные скобки или вообще без скобок, например `1.5em 1em 0 2em`, `Helvetica, Arial, sans-serif` или `[col1-start]`.
- [Карты](/values/maps), которые связывают значения с ключами, например `("background": red, "foreground": pink)`.

### Операции {#operations}

Sass определяет синтаксис для ряда операций:

- [`==` и `!=`](/operators/equality) используются для проверки, являются ли два значения одинаковыми.
- [`+`, `-`, `*`, `/` и `%`](/operators/numeric) имеют своё обычное математическое значение для чисел, со специальным поведением для единиц измерения, которое соответствует использованию единиц в научной математике.
- [`<`, `<=`, `>` и `>=`](/operators/relational) проверяют, является ли одно число больше или меньше другого.
- [`and`, `or` и `not`](/operators/boolean) имеют обычное логическое поведение. Sass считает каждое значение «истинным», за исключением `false` и `null`.
- [`+`, `-` и `/`](/operators/string) могут использоваться для конкатенации строк.
- [`(` и `)`](/operators#parentheses) могут использоваться для явного управления порядком приоритета операций.

### Другие выражения {#other-expressions}

- [Переменные](/variables), например `$var`.
- [Вызовы функций](/at-rules/function), например `nth($list, 1)` или `var(--main-bg-color)`, которые могут вызывать функции основной библиотеки Sass или пользовательские функции, или могут быть скомпилированы непосредственно в CSS.
- [Специальные функции](special-functions), например `calc(1px + 100%)` или `url(http://myapp.com/assets/logo.png)`, которые имеют свои уникальные правила парсинга.
- [Родительский селектор](/style-rules/parent-selector), `&`.
- Значение `!important`, которое анализируется как строка без кавычек.
