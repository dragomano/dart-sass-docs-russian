> Числа в Sass состоят из двух компонентов: самого числа и его единиц измерения. Например, в `16px` число — это `16`, а единица измерения — `px`. Числа могут не иметь единиц измерения, а также могут иметь составные единицы. Подробнее см. раздел [Единицы измерения](#units) ниже.

=== "SCSS"

    ```scss
    @debug 100; // 100
    @debug 0.8; // 0.8
    @debug 16px; // 16px
    @debug 5px * 2px; // 10px*px (читается как "квадратных пикселей")
    ```

=== "SASS"

    ```sass
    @debug 100  // 100
    @debug 0.8  // 0.8
    @debug 16px  // 16px
    @debug 5px * 2px  // 10px*px (читается как "квадратных пикселей")
    ```

Числа в Sass поддерживают те же форматы, что и CSS-числа, включая [экспоненциальную запись](https://ru.wikipedia.org/wiki/%D0%AD%D0%BA%D1%81%D0%BF%D0%BE%D0%BD%D0%B5%D0%BD%D1%86%D0%B8%D0%B0%D0%BB%D1%8C%D0%BD%D0%B0%D1%8F_%D0%B7%D0%B0%D0%BF%D0%B8%D1%81%D1%8C), которая записывается с буквой `e` между числом и его степенью десяти. Поскольку поддержка экспоненциальной записи в браузерах исторически была нестабильной, Sass всегда компилирует её в полностью развёрнутые числа.

=== "SCSS"

    ```scss
    @debug 5.2e3; // 5200
    @debug 6e-2; // 0.06
    ```

=== "SASS"

    ```sass
    @debug 5.2e3  // 5200
    @debug 6e-2  // 0.06
    ```

!!! note "Примечание"

    Sass не различает целые числа и десятичные, поэтому, например, `math.div(5, 2)` возвращает `2.5`, а не `2`. Это то же поведение, что и в JavaScript, но отличается от многих других языков программирования.

## Единицы измерения {#units}

Sass обладает мощной поддержкой манипуляций с единицами измерения, основанной на том, как работают [вычисления с физическими единицами](https://ru.wikipedia.org/wiki/%D0%95%D0%B4%D0%B8%D0%BD%D0%B8%D1%86%D1%8B_%D1%84%D0%B8%D0%B7%D0%B8%D1%87%D0%B5%D1%81%D0%BA%D0%B8%D1%85_%D0%B2%D0%B5%D0%BB%D0%B8%D1%87%D0%B8%D0%BD) в реальном мире. Когда два числа умножаются, их единицы измерения также умножаются. Когда одно число делится на другое, результат получает единицы числителя от первого числа и единицы знаменателя — от второго. Число может иметь любое количество единиц измерения в числителе и/или знаменателе.

=== "SCSS"

    ```scss
    @use 'sass:math';

    @debug 4px * 6px; // calc(24px * 1px) (читается как "квадратных пикселей")
    @debug math.div(5px, 2s); // calc(2.5px / 1s) (читается как "пикселей в секунду")

    // calc(3.125px * 1deg / 1s / 1em) (читается как "пиксель-градусов на секунду-em")
    @debug 5px * math.div(math.div(30deg, 2s), 24em);

    $degrees-per-second: math.div(20deg, 1s);
    @debug $degrees-per-second; // calc(20deg / 1s)
    @debug math.div(1, $degrees-per-second); // calc(0.05s / 1deg)
    ```

=== "SASS"

    ```sass
    @use 'sass:math'

    @debug 4px * 6px  // calc(24px * 1px) (читается как "квадратных пикселей")
    @debug math.div(5px, 2s)  // calc(2.5px / 1s) (читается как "пикселей в секунду")

    // calc(3.125px * 1deg / 1s / 1em) (читается как "пиксель-градусов на секунду-em")
    @debug 5px * math.div(math.div(30deg, 2s), 24em)

    $degrees-per-second: math.div(20deg, 1s)
    @debug $degrees-per-second  // calc(20deg / 1s)
    @debug math.div(1, $degrees-per-second)  // calc(0.05s / 1deg)
    ```

!!! tip "Совет"

    В CSS числа со сложными единицами измерения могут быть представлены только в `calc()` и других математических выражениях, поэтому Sass выводит такие числа как выражения `calc()`. Обычные CSS-свойства не принимают сложные единицы измерения, но они могут быть полезны в [пользовательских свойствах](https://developer.mozilla.org/en-US/docs/Web/CSS/Guides/Cascading_variables/Using_custom_properties) или [функциях](https://developer.mozilla.org/en-US/docs/Web/CSS/Guides/Custom_functions_and_mixins/Using_custom_functions).

Sass автоматически преобразует совместимые единицы измерения друг в друга, хотя выбор единицы для результата зависит от того, какую реализацию Sass вы используете. Если вы попытаетесь скомбинировать несовместимые единицы, например `1in + 1em`, Sass выдаст ошибку.

=== "SCSS"

    ```scss
    // CSS определяет один дюйм как 96 пикселей.
    @debug 1in + 6px; // 102px или 1.0625in

    @debug 1in + 1s;
    //     ^^^^^^^^
    // Ошибка: Несовместимые единицы s и in.
    ```

=== "SASS"

    ```sass
    // CSS определяет один дюйм как 96 пикселей.
    @debug 1in + 6px  // 102px или 1.0625in

    @debug 1in + 1s
    //     ^^^^^^^^
    // Ошибка: Несовместимые единицы s и in.
    ```

Как и в вычислениях с физическими единицами, если в числителе содержатся единицы, совместимые с единицами в знаменателе (например, `math.div(96px, 1in)`), они сокращаются. Это упрощает определение соотношения, которое можно использовать для преобразования между единицами измерения. В примере ниже мы устанавливаем желаемую скорость как одна секунда на 50 пикселей, а затем умножаем её на количество пикселей, которые покрывает переход, чтобы получить время, которое он должен занять.

<div class="grid" markdown>

=== "SCSS"

    ```scss
    @use 'sass:math';

    $transition-speed: math.div(1s, 50px);

    @mixin move($left-start, $left-stop) {
        position: absolute;
        left: $left-start;
        transition: left ($left-stop - $left-start) * $transition-speed;

        &:hover {
        left: $left-stop;
        }
    }

    .slider {
        @include move(10px, 120px);
    }
    ```

=== "SASS"

    ```sass
    @use 'sass:math'

    $transition-speed: math.div(1s, 50px)

    @mixin move($left-start, $left-stop)
        position: absolute
        left: $left-start
        transition: left ($left-stop - $left-start) * $transition-speed

        &:hover
        left: $left-stop

    .slider
        @include move(10px, 120px)
    ```

```css title="CSS"
.slider {
  position: absolute;
  left: 10px;
  transition: left 2.2s;
}
.slider:hover {
  left: 120px;
}
```

!!! tip "Совет"

    Если ваша арифметика даёт неправильную единицу измерения, вам, вероятно, нужно проверить свои вычисления. Возможно, вы забыли указать единицы для величины, которая должна их иметь! Поддержание чистоты единиц измерения позволяет Sass выдавать полезные ошибки, когда что-то идёт не так.

    Особенно следует избегать использования интерполяции вроде `#{$number}px`. Это на самом деле не создаёт число! Это создаёт [строку без кавычек](../values/strings#unquoted), которая *выглядит* как число, но не будет работать ни с какими [числовыми операциями](../operators/numeric), ни с [функциями](../modules/math). Постарайтесь сделать свою математику чистой по единицам, чтобы `$number` уже имел единицу `px`, или напишите `$number * 1px`.

!!! note "Примечание"

    Проценты в Sass работают так же, как и любая другая единица измерения. Они *не* взаимозаменяемы с десятичными дробями, потому что в CSS десятичные дроби и проценты означают разные вещи. Например, `50%` — это число с `%` в качестве единицы измерения, и Sass считает его отличным от числа `0.5`.

    Вы можете преобразовывать между десятичными дробями и процентами, используя арифметику единиц. `math.div($percentage, 100%)` вернёт соответствующую десятичную дробь, а `$decimal * 100%` вернёт соответствующий процент. Также можно использовать [функцию `math.percentage()`](../modules/math#percentage) как более явный способ записи `$decimal * 100%`.

## Точность {#precision}

Числа в Sass представлены внутренне как 64-битные значения с плавающей точкой. Они поддерживают до 10 цифр точности после десятичной точки при сериализации в CSS и для целей сравнения на равенство. Это означает несколько разных вещей:

* Только первые десять цифр числа после десятичной точки будут включены в сгенерированный CSS.

* Операции вроде [`==`](../operators/equality) и [`>=`](../operators/relational) будут считать два числа эквивалентными, если они совпадают до десятой цифры после десятичной точки.

* Если число отличается от целого меньше чем на `0.0000000001`, оно считается целым для целей функций вроде [`list.nth()`](../modules/list#nth), которые требуют целочисленных аргументов.

=== "SCSS"

    ```scss
    @debug 0.012345678912345; // 0.0123456789
    @debug 0.01234567891 == 0.01234567899; // true
    @debug 1.00000000009; // 1
    @debug 0.99999999991; // 1
    ```

=== "SASS"

    ```sass
    @debug 0.012345678912345  // 0.0123456789
    @debug 0.01234567891 == 0.01234567899  // true
    @debug 1.00000000009  // 1
    @debug 0.99999999991  // 1
    ```

!!! note "Примечание"

    Числа округляются до 10 цифр точности *лениво*, когда они используются в месте, где точность имеет значение. Это означает, что математические функции будут работать с полным значением числа внутренне, чтобы избежать накопления дополнительных ошибок округления.
