Sass имеет встроенную поддержку цветовых значений. Как и цвета CSS, каждый цвет представляет точку в определённом цветовом пространстве, таком как `rgb` или `lab`. Цвета Sass могут быть записаны как шестнадцатеричные коды (`#f2ece4` или `#b37399aa`), [названия цветов CSS](https://developer.mozilla.org/en-US/docs/Web/CSS/color_value#Color_keywords) (`midnightblue`, `transparent`) или цветовые функции, такие как [`rgb()`](../modules#rgb), [`lab()`](../modules#lab) или [`color()`](../modules#color).

=== "SCSS"

    ```scss
    @debug #f2ece4; // #f2ece4
    @debug #b37399aa; // rgba(179, 115, 153, 67%)
    @debug midnightblue; // #191970
    @debug rgb(204 102 153); // #c69
    @debug lab(32.4% 38.4 -47.7 / 0.7); // lab(32.4% 38.4 -47.7 / 0.7)
    @debug color(display-p3 0.597 0.732 0.576); // color(display-p3 0.597 0.732 0.576)
    ```

=== "SASS"

    ```sass
    @debug #f2ece4 // #f2ece4
    @debug #b37399aa // rgba(179, 115, 153, 67%)
    @debug midnightblue // #191970
    @debug rgb(204 102 153) // #c69
    @debug lab(32.4% 38.4 -47.7 / 0.7) // lab(32.4% 38.4 -47.7 / 0.7)
    @debug color(display-p3 0.597 0.732 0.576) // color(display-p3 0.597 0.732 0.576)
    ```

## Цветовые пространства {#color-spaces}

Sass поддерживает тот же набор цветовых пространств, что и CSS. Цвет Sass всегда будет выводиться в том же цветовом пространстве, в котором он был написан, если только он не находится в [устаревшем цветовом пространстве](#legacy-color-spaces) или вы не конвертируете его в другое пространство с помощью [`color.to-space()`](../modules/color#to-space). Все остальные цветовые функции в Sass всегда будут возвращать цвет в тех же пространствах, что и исходный цвет, даже если функция внесла изменения в этот цвет в другом
пространстве.

Хотя каждое цветовое пространство имеет границы гаммы, которые оно ожидает для своих каналов, Sass может представлять значения вне гаммы для любого цветового пространства. Это позволяет цвету из широкогаммутного пространства безопасно конвертироваться в узкогаммутное пространство и обратно без потери информации.

!!! tip "Совет"

    CSS требует, чтобы некоторые цветовые функции обрезали свои входные каналы. Например, `rgb(500 0 0)` обрезает свой красный канал, чтобы он находился в пределах [0, 255], и поэтому эквивалентен `rgb(255 0 0)`, даже несмотря на то, что `rgb(500 0 0)` является отдельным значением, которое Sass может представить. Вы всегда можете использовать функцию [`color.change()`](../modules/color#change) в Sass, чтобы установить значение вне гаммы для любого пространства.

Ниже приведен полный список всех цветовых пространств, которые поддерживает Sass. Вы можете узнать об этих пространствах [на MDN](https://developer.mozilla.org/en-US/docs/Glossary/Color_space).

| Пространство | Синтаксис | Каналы [min, max] |
|--------------|-----------|-------------------|
| `rgb`* | `rgb(102 51 153)`<br>`#663399`<br>`rebeccapurple` | red [0, 255]; green [0, 255]; blue [0, 255] |

| `srgb` | `color(srgb 0.4 0.2 0.6)` | red [0, 1]; green [0, 1]; blue [0, 1] |
| `srgb-linear` | `color(srgb-linear 0.133 0.033 0.319)` | red [0, 1]; green [0, 1]; blue [0, 1] |

## Отсутствующие каналы {#missing-channels}

Цвета в CSS и Sass могут иметь «отсутствующие каналы», которые записываются как `none` и представляют канал, значение которого неизвестно или не влияет на то, как цвет отображается. Например, вы можете написать `hsl(none 0% 50%)`, потому что тон не имеет значения, если насыщенность равна `0%`. В большинстве случаев отсутствующие каналы просто обрабатываются как нулевые значения, но они иногда встречаются:

* Если вы смешиваете цвета вместе, либо как часть интерполяции CSS для чего-то вроде анимации, либо используя функцию [`color.mix()`](../modules/color#mix) в Sass, отсутствующие каналы всегда принимают значение другого цвета для этого канала, если это возможно.

* Если вы конвертируете цвет с отсутствующим каналом в другое пространство, которое имеет аналогичный канал, этот канал будет установлен в `none` после завершения конвертации.

Хотя [`color.channel()`](../modules/color#channel) вернет 0 для отсутствующих каналов, вы всегда можете проверить их наличие с помощью [`color.is-missing()`](../modules/color#is-missing).

=== "SCSS"

    ```scss
    @use 'sass:color';

    $grey: hsl(none 0% 50%);

    @debug color.mix($grey, blue, $method: hsl); // hsl(240, 50%, 50%)
    @debug color.to-space($grey, lch); // lch(53.3889647411% 0 none)
    ```

=== "SASS"

    ```sass
    @use 'sass:color'

    $grey: hsl(none 0% 50%)

    @debug color.mix($grey, blue, $method: hsl) // hsl(240, 50%, 50%)
    @debug color.to-space($grey, lch) // lch(53.3889647411% 0 none)
    ```

### Неэффективные каналы {#powerless-channels}

Цветовой канал считается «неэффективным» при определённых обстоятельствах, когда его значение не влияет на то, как цвет отображается на экране. Спецификация CSS требует, чтобы при конвертации цвета в новое пространство все неэффективные каналы заменялись на `none`. Sass делает это во всех случаях, за исключением конвертации в устаревшие пространства, чтобы гарантировать, что конвертация в устаревшее пространство всегда создает цвет, совместимый со старыми браузерами.

Для получения более подробной информации о неэффективных каналах см. [`color.is-powerless()`](../modules/color#is-powerless).

## Устаревшие цветовые пространства {#legacy-color-spaces}

Исторически CSS и Sass поддерживали только стандартную RGB гамму и только функции `rgb`, `hsl` и `hwb` для определения цветов. Поскольку в то время все цвета использовали одну и ту же гамму, каждая цветовая функция работала с каждым цветом независимо от его цветового пространства. Sass всё ещё сохраняет это поведение, но только для старых функций и только для цветов в этих трех «устаревших» цветовых пространствах. Тем не менее, по-прежнему является хорошей практикой явно указывать `$space`, в котором вы хотите работать, при использовании цветовых функций.

Sass также свободно конвертирует между различными устаревшими цветовыми пространствами при конвертации устаревших цветовых значений в CSS. Это всегда безопасно, потому что все они используют одну и ту же базовую цветовую модель, и это помогает гарантировать, что Sass выводит цвета в максимально совместимом формате.

## Цветовые функции {#color-functions}

Sass поддерживает множество полезных [цветовых функций](../modules/color), которые можно использовать для создания новых цветов на основе существующих путём [смешивания цветов](../modules/color#mix) или [масштабирования значений их каналов](../modules/color#scale). При вызове цветовых функций цветовые пространства всегда должны быть записаны как строки без кавычек, чтобы соответствовать CSS, в то время как названия каналов должны быть записаны как строки в кавычках, чтобы каналы вроде `"red"` не интерпретировались как цветовые значения.

!!! note "Примечание"

    Цветовые функции Sass могут автоматически конвертировать цвета между пространствами, что упрощает выполнение преобразований в перцептуально-однородных цветовых пространствах, таких как Oklch. Но они *всегда* возвращают цвет в том же пространстве, которое вы им дали, если только вы явно не вызовете [`color.to-space()`](../modules/color#to-space) для его конвертации.

=== "SCSS"

    ```scss
    @use 'sass:color';

    $venus: #998099;

    @debug color.scale($venus, $lightness: +15%, $space: oklch);
    // rgb(170.1523703626, 144.612080603, 170.1172627174)
    @debug color.mix($venus, midnightblue, $method: oklch);
    // rgb(95.9363315581, 74.5687109346, 133.2082569526)
    ```

=== "SASS"

    ```sass
    @use 'sass:color'

    $venus: #998099

    @debug color.scale($venus, $lightness: +15%, $space: oklch)
    // rgb(170.1523703626, 144.612080603, 170.1172627174)
    @debug color.mix($venus, midnightblue, $method: oklch)
    // rgb(95.9363315581, 74.5687109346, 133.2082569526)
    ```
