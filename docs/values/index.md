> Sass оперирует различными типами данных, значительная часть которых совпадает с тем, что уже существует в CSS. Каждое [выражение](../syntax/structure#expressions) создаёт значение, [переменные](../variables) хранят значения.

* [Числа](../values/numbers), которые могут иметь или не иметь единицы измерения, например `12` или `100px`.

* [Строки](../values/strings), которые могут быть в кавычках или без них, например `"Helvetica Neue"` или `bold`.

* [Цвета](../values/colors), на которые можно ссылаться по их шестнадцатеричному представлению или по имени, например `#c6538c` или `blue`, или возвращаемые из функций, например `rgb(107, 113, 127)` или `hsl(210, 100%, 20%)`.

* [Списки значений](../values/lists), которые могут разделяться пробелами или запятыми и могут быть заключены в квадратные скобки или не иметь скобок вообще, например `1.5em 1em 0 2em`, `Helvetica, Arial, sans-serif` или `[col1-start]`.

Некоторые другие типы специфичны для Sass:

* [Булевы](../values/booleans) значения `true` и `false`.

* Единственное значение [`null`](../values/null).

* [Карты](../values/maps), которые связывают значения с ключами, например `("background": red, "foreground": pink)`.

* [Ссылки на функции](../values/functions), возвращаемые функцией [`get-function()`](../modules/meta#get-function) и вызываемые с помощью [`call()`](../modules/meta#call).
