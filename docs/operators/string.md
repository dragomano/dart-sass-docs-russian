> Sass поддерживает несколько операторов, которые генерируют [строки](../values/strings):

* `<выражение> + <выражение>` возвращает строку, содержащую значения обоих выражений. Если хотя бы одно из значений является [строкой в кавычках](../values/strings#quoted), результат будет в кавычках; в противном случае он будет без кавычек.
* `<выражение> - <выражение>` возвращает строку без кавычек, содержащую значения обоих выражений, разделённые `-`. Это устаревший оператор, и вместо него обычно следует использовать [интерполяцию](../interpolation).

=== "SCSS"

    ```scss
    @debug "Helvetica" + " Neue"; // "Helvetica Neue"
    @debug sans- + serif; // sans-serif
    @debug sans - serif; // sans-serif
    ```

=== "SASS"

    ```sass
    @debug "Helvetica" + " Neue" // "Helvetica Neue"
    @debug sans- + serif // sans-serif
    @debug sans - serif // sans-serif
    ```

Эти операторы работают не только со строками! Их можно использовать с любыми значениями, которые могут быть записаны в CSS, за несколькими исключениями:

* Числа нельзя использовать в качестве левого операнда, потому что у них есть [собственные операторы](../operators/numeric).
* Цвета нельзя использовать в качестве левого операнда, потому что раньше у них были [собственные операторы](../operators).

=== "SCSS"

    ```scss
    @debug "Прошло времени: " + 10s; // "Прошло времени: 10s";
    @debug true + " является логическим значением"; // "true является логическим значением";
    ```

=== "SASS"

    ```sass
    @debug "Прошло времени: " + 10s // "Прошло времени: 10s";
    @debug true + " является логическим значением" // "true является логическим значением";
    ```

!!! note "Примечание"

    Часто использование [интерполяции](../interpolation) для создания строк является более чистым и понятным подходом, чем использование этих операторов.

## Унарные операторы {#unary-operators}

По историческим причинам Sass также поддерживает `/` и `-` как унарные операторы, которые принимают только одно значение:

* `/<выражение>` возвращает строку без кавычек, начинающуюся с `/` и за которой следует значение выражения.
* `-<выражение>` возвращает строку без кавычек, начинающуюся с `-` и за которой следует значение выражения.

=== "SCSS"

    ```scss
    @debug / 15px; // /15px
    @debug - moz; // -moz
    ```

=== "SASS"

    ```sass
    @debug / 15px // /15px
    @debug - moz // -moz
    ```
