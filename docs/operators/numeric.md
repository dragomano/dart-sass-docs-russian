> Sass поддерживает стандартный набор математических операторов для работы с [числами](../values/numbers). Они автоматически выполняют преобразование между совместимыми единицами измерения.

* `<выражение> + <выражение>` складывает значение первого [выражения](../syntax/structure#expressions) со вторым.
* `<выражение> - <выражение>` вычитает значение второго [выражения](../syntax/structure#expressions) из первого.
* `<выражение> * <выражение>` умножает значение первого [выражения](../syntax/structure#expressions) на второе.
* `<выражение> % <выражение>` возвращает остаток от деления значения первого [выражения](../syntax/structure#expressions) на второе. Это известно как [оператор *деления по модулю*](https://en.wikipedia.org/wiki/Modulo_operation).

=== "SCSS"

    ```scss
    @debug 10s + 15s; // 25s
    @debug 1in - 10px; // 0.8958333333in
    @debug 5px * 3px; // 15px*px
    @debug 1in % 9px; // 0.0625in
    ```

=== "SASS"

    ```sass
    @debug 10s + 15s // 25s
    @debug 1in - 10px // 0.8958333333in
    @debug 5px * 3px // 15px*px
    @debug 1in % 9px // 0.0625in
    ```

Безразмерные числа могут использоваться вместе с числами в любых единицах.

=== "SCSS"

    ```scss
    @debug 100px + 50; // 150px
    @debug 4s * 10; // 40s
    ```

=== "SASS"

    ```sass
    @debug 100px + 50 // 150px
    @debug 4s * 10 // 40s
    ```

Числа с несовместимыми единицами нельзя использовать с операциями сложения, вычитания или модуля.

=== "SCSS"

    ```scss
    @debug 100px + 10s;
    //     ^^^^^^^^^^^
    // Ошибка: несовместимые единицы измерения px и s.
    ```

=== "SASS"

    ```sass
    @debug 100px + 10s
    //     ^^^^^^^^^^^
    // Ошибка: несовместимые единицы измерения px и s.
    ```

## Унарные операторы {#unary-operators}

Вы также можете использовать `+` и `-` как унарные операторы, которые принимают только одно значение:

* `+<выражение>` возвращает значение выражения без изменений.
* `-<выражение>` возвращает отрицательную версию значения выражения.

=== "SCSS"

    ```scss
    @debug +(5s + 7s); // 12s
    @debug -(50px + 30px); // -80px
    @debug -(10px - 15px); // 5px
    ```

=== "SASS"

    ```sass
    @debug +(5s + 7s) // 12s
    @debug -(50px + 30px) // -80px
    @debug -(10px - 15px) // 5px
    ```

!!! note "Примечание"

    Поскольку `-` может означать как вычитание, так и унарное отрицание, это может вызвать путаницу в списках, разделённых пробелами. Для безопасности:

    * Всегда ставьте пробелы по обе стороны от `-` при вычитании.
    * Ставьте пробел перед `-`, но не после для отрицательного числа или унарного отрицания.
    * Заключайте унарное отрицание в круглые скобки, если оно находится в списке, разделённом пробелами.

    Различные значения `-` в Sass имеют приоритет в следующем порядке:

    1. `-` как часть идентификатора. Единственное исключение — единицы измерения; Sass обычно позволяет использовать любой допустимый идентификатор в качестве идентификатора, но единицы измерения не могут содержать дефис, за которым следует цифра.
    2. `-` между выражением и числовым литералом без пробелов, что интерпретируется как вычитание.
    3. `-` в начале числового литерала, что интерпретируется как отрицательное число.
    4. `-` между двумя числами независимо от пробелов, что интерпретируется как вычитание.
    5. `-` перед значением, отличным от числового литерала, что интерпретируется как унарное отрицание.

    === "SCSS"

        ```scss
        @debug a-1; // a-1
        @debug 5px-3px; // 2px
        @debug 5-3; // 2
        @debug 1 -2 3; // 1 -2 3

        $number: 2;
        @debug 1 -$number 3; // -1 3
        @debug 1 (-$number) 3; // 1 -2 3
        ```

    === "SASS"

        ```sass
        @debug a-1 // a-1
        @debug 5px-3px // 2px
        @debug 5-3 // 2
        @debug 1 -2 3 // 1 -2 3

        $number: 2
        @debug 1 -$number 3 // -1 3
        @debug 1 (-$number) 3 // 1 -2 3
        ```

## Деление {#division}

В отличие от других математических операций, деление в Sass выполняется с помощью функции [`math.div()`](../modules/math#div). Хотя во многих языках программирования `/` используется как оператор деления, в CSS этот символ выступает разделителем (например, `font: 15px/32px` или `hsl(120 100% 50% / 0.8)`). Тем не менее Sass поддерживает использование `/` как оператора деления, но это поведение устарело и [будет удалено](../breaking-changes/slash-div) в одной из будущих версий.

### Значения, разделённые слэшем {#slash-separated-values}

Пока Sass всё ещё поддерживает `/` как оператор деления, ему необходимо различать случаи, когда `/` выступает разделителем, а когда — операцией деления. Для этого, если два числа разделены `/`, Sass выведет результат как значения, разделённые слэшем, а не как результат деления, если только не выполняется одно из следующих условий:

* Любое из выражений является чем‑то, отличным от числового литерала.
* Результат сохраняется в переменную или возвращается функцией.
* Операция заключена в круглые скобки, за исключением случаев, когда эти скобки находятся снаружи списка, содержащего операцию.
* Результат используется как часть другой операции (отличной от `/`).
* Результат возвращается [вычислением](../values/calculations).

Вы можете использовать [`list.slash()`](../modules/list#slash), чтобы принудительно интерпретировать `/` как разделитель.

=== "SCSS"

    ```scss
    @use "sass:list";

    @debug 15px / 30px; // 15px/30px
    @debug (10px + 5px) / 30px; // 0.5
    @debug list.slash(10px + 5px, 30px); // 15px/30px

    $result: 15px / 30px;
    @debug $result; // 0.5

    @function fifteen-divided-by-thirty() {
      @return 15px / 30px;
    }
    @debug fifteen-divided-by-thirty(); // 0.5

    @debug (15px/30px); // 0.5
    @debug (bold 15px/30px sans-serif); // bold 15px/30px sans-serif
    @debug 15px/30px + 1; // 1.5
    ```

=== "SASS"

    ```sass
    @use "sass:list";

    @debug 15px / 30px // 15px/30px
    @debug (10px + 5px) / 30px // 0.5
    @debug list.slash(10px + 5px, 30px) // 15px/30px

    $result: 15px / 30px
    @debug $result // 0.5

    @function fifteen-divided-by-thirty()
      @return 15px / 30px

    @debug fifteen-divided-by-thirty() // 0.5

    @debug (15px/30px) // 0.5
    @debug (bold 15px/30px sans-serif) // bold 15px/30px sans-serif
    @debug 15px/30px + 1 // 1.5
    ```

## Единицы измерения {#units}

Sass обладает мощной поддержкой манипуляций с единицами измерения, основанной на том, как работают [вычисления с физическими единицами](https://ru.wikipedia.org/wiki/%D0%95%D0%B4%D0%B8%D0%BD%D0%B8%D1%86%D1%8B_%D1%84%D0%B8%D0%B7%D0%B8%D1%87%D0%B5%D1%81%D0%BA%D0%B8%D1%85_%D0%B2%D0%B5%D0%BB%D0%B8%D1%87%D0%B8%D0%BD) в реальном мире. Когда два числа умножаются, их единицы измерения также умножаются. Когда одно число делится на другое, результат получает единицы числителя от первого числа и единицы знаменателя — от второго. Число может иметь любое количество единиц измерения в числителе и/или знаменателе.

=== "SCSS"

    ```scss
    @use 'sass:math';

    @debug 4px * 6px; // calc(24px * 1px) (читается как "квадратных пикселей")
    @debug math.div(5px, 2s); // calc(2.5px / 1s) (читается как "пикселей в секунду")

    // calc(3.125px * 1deg / 1s / 1em) (читается как "пиксель-градусов на секунду-em")
    @debug 5px * math.div(math.div(30deg, 2s), 24em);

    $degrees-per-second: math.div(20deg, 1s);
    @debug $degrees-per-second; // calc(20deg / 1s)
    @debug math.div(1, $degrees-per-second); // calc(0.05s / 1deg)
    ```

=== "SASS"

    ```sass
    @use 'sass:math'

    @debug 4px * 6px  // calc(24px * 1px) (читается как "квадратных пикселей")
    @debug math.div(5px, 2s)  // calc(2.5px / 1s) (читается как "пикселей в секунду")

    // calc(3.125px * 1deg / 1s / 1em) (читается как "пиксель-градусов на секунду-em")
    @debug 5px * math.div(math.div(30deg, 2s), 24em)

    $degrees-per-second: math.div(20deg, 1s)
    @debug $degrees-per-second  // calc(20deg / 1s)
    @debug math.div(1, $degrees-per-second)  // calc(0.05s / 1deg)
    ```

!!! tip "Совет"

    В CSS числа со сложными единицами измерения могут быть представлены только в `calc()` и других математических выражениях, поэтому Sass выводит такие числа как выражения `calc()`. Обычные CSS-свойства не принимают сложные единицы измерения, но они могут быть полезны в [пользовательских свойствах](https://developer.mozilla.org/en-US/docs/Web/CSS/Guides/Cascading_variables/Using_custom_properties) или [функциях](https://developer.mozilla.org/en-US/docs/Web/CSS/Guides/Custom_functions_and_mixins/Using_custom_functions).

Sass автоматически преобразует совместимые единицы измерения друг в друга, хотя выбор единицы для результата зависит от того, какую реализацию Sass вы используете. Если вы попытаетесь скомбинировать несовместимые единицы, например `1in + 1em`, Sass выдаст ошибку.

=== "SCSS"

    ```scss
    // CSS определяет один дюйм как 96 пикселей.
    @debug 1in + 6px; // 102px или 1.0625in

    @debug 1in + 1s;
    //     ^^^^^^^^
    // Ошибка: Несовместимые единицы s и in.
    ```

=== "SASS"

    ```sass
    // CSS определяет один дюйм как 96 пикселей.
    @debug 1in + 6px  // 102px или 1.0625in

    @debug 1in + 1s
    //     ^^^^^^^^
    // Ошибка: Несовместимые единицы s и in.
    ```

Как и в вычислениях с физическими единицами, если в числителе содержатся единицы, совместимые с единицами в знаменателе (например, `math.div(96px, 1in)`), они сокращаются. Это упрощает определение соотношения, которое можно использовать для преобразования между единицами измерения. В примере ниже мы устанавливаем желаемую скорость как одна секунда на 50 пикселей, а затем умножаем её на количество пикселей, которые покрывает переход, чтобы получить время, которое он должен занять.

<div class="grid" markdown>

=== "SCSS"

    ```scss
    @use 'sass:math';

    $transition-speed: math.div(1s, 50px);

    @mixin move($left-start, $left-stop) {
        position: absolute;
        left: $left-start;
        transition: left ($left-stop - $left-start) * $transition-speed;

        &:hover {
        left: $left-stop;
        }
    }

    .slider {
        @include move(10px, 120px);
    }
    ```

=== "SASS"

    ```sass
    @use 'sass:math'

    $transition-speed: math.div(1s, 50px)

    @mixin move($left-start, $left-stop)
        position: absolute
        left: $left-start
        transition: left ($left-stop - $left-start) * $transition-speed

        &:hover
        left: $left-stop

    .slider
        @include move(10px, 120px)
    ```

```css title="CSS"
.slider {
  position: absolute;
  left: 10px;
  transition: left 2.2s;
}
.slider:hover {
  left: 120px;
}
```

!!! tip "Совет"

    Если ваша арифметика даёт неправильную единицу измерения, вам, вероятно, нужно проверить свои вычисления. Возможно, вы забыли указать единицы для величины, которая должна их иметь! Поддержание чистоты единиц измерения позволяет Sass выдавать полезные ошибки, когда что-то идёт не так.

    Особенно следует избегать использования интерполяции вроде `#{$number}px`. Это на самом деле не создаёт число! Это создаёт [строку без кавычек](../values/strings#unquoted), которая *выглядит* как число, но не будет работать ни с какими [числовыми операциями](../operators/numeric), ни с [функциями](../modules/math). Постарайтесь сделать свою математику чистой по единицам, чтобы `$number` уже имел единицу `px`, или напишите `$number * 1px`.

!!! note "Примечание"

    Проценты в Sass работают так же, как и любая другая единица измерения. Они *не* взаимозаменяемы с десятичными дробями, потому что в CSS десятичные дроби и проценты означают разные вещи. Например, `50%` — это число с `%` в качестве единицы измерения, и Sass считает его отличным от числа `0.5`.

    Вы можете преобразовывать между десятичными дробями и процентами, используя арифметику единиц. `math.div($percentage, 100%)` вернёт соответствующую десятичную дробь, а `$decimal * 100%` вернёт соответствующий процент. Также можно использовать [функцию `math.percentage()`](../modules/math#percentage) как более явный способ записи `$decimal * 100%`.
