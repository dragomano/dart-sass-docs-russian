> Sass поддерживает несколько полезных `операторов` для работы с различными значениями. К ним относятся стандартные математические операторы, такие как `+` и `*`, а также операторы для различных других типов:

- [`==` и `!=`](../operators/equality) используются для проверки, являются ли два значения одинаковыми.
- [`+`, `-`, `*`, `/` и `%`](../operators/numeric) имеют своё обычное математическое значение для чисел, со специальным поведением для единиц измерения, которое соответствует использованию единиц в научной математике.
- [`<`, `<=`, `>` и `>=`](../operators/relational) проверяют, является ли одно число больше или меньше другого.
- [`and`, `or` и `not`](../operators/boolean) имеют обычное логическое поведение. Sass считает каждое значение «истинным», за исключением `false` и `null`.
- [`+`, `-` и `/`](../operators/string) могут использоваться для конкатенации строк.
- [`(` и `)`](../operators#parentheses) могут использоваться для явного управления порядком приоритета операций.

!!! info "Информация"

    На раннем этапе истории Sass была добавлена поддержка математических операций с [цветами](../values/colors). Эти операции выполнялись отдельно для каждого RGB-канала цветов, поэтому сложение двух цветов давало цвет, у которого красный канал равен сумме их красных каналов, и так далее.

    Такое поведение оказалось не очень полезным, поскольку поканальная арифметика RGB плохо соответствовала тому, как люди воспринимают цвет. Были добавлены [функции для работы с цветом](../modules/color), которые гораздо полезнее, а операции с цветами были объявлены устаревшими. Они всё ещё поддерживаются в LibSass и Ruby Sass, но выдают предупреждения, и пользователям настоятельно рекомендуется избегать их использования.

## Порядок операций {#order-of-operations}

В Sass действует довольно стандартный [порядок операций](https://ru.wikipedia.org/wiki/%D0%9F%D1%80%D0%B8%D0%BE%D1%80%D0%B8%D1%82%D0%B5%D1%82_%D0%BE%D0%BF%D0%B5%D1%80%D0%B0%D1%86%D0%B8%D0%B8) — от наивысшего приоритета к наименьшему:

1. Унарные операторы [`not`](../operators/boolean), [`+`, `-`](../operators/numeric#unary-operators) и [`/`](../operators/string#unary-operators).
2. Операторы [`*`, `/` и `%`](../operators/numeric).
3. Операторы [`+` и `-`](../operators/numeric).
4. Операторы [`>`, `>=`, `<` и `<=`](../operators/relational).
5. Операторы [`==` и `!=`](../operators/equality).
6. Оператор [`and`](../operators/boolean).
7. Оператор [`or`](../operators/boolean).
8. Оператор [`=`](#single-equals), когда он доступен.

=== "SCSS"

    ```scss
    @debug 1 + 2 * 3 == 1 + (2 * 3); // true
    @debug true or false and false == true or (false and false); // true
    ```

=== "SASS"

    ```sass
    @debug 1 + 2 * 3 == 1 + (2 * 3) // true
    @debug true or false and false == true or (false and false) // true
    ```

### Скобки {#parentheses}

Вы можете явно управлять порядком операций с помощью скобок. Операция внутри скобок всегда выполняется раньше любых операций вне их. Скобки можно даже вкладывать друг в друга, и в этом случае сначала будут вычислены самые внутренние скобки.

=== "SCSS"

    ```scss
    @debug (1 + 2) * 3; // 9
    @debug ((1 + 2) * 3 + 4) * 5; // 65
    ```

=== "SASS"

    ```sass
    @debug (1 + 2) * 3 // 9
    @debug ((1 + 2) * 3 + 4) * 5 // 65
    ```

## Одинарное равенство {#single-equals}

Sass поддерживает специальный оператор `=`, который разрешён только в аргументах функций и просто создаёт [строку без кавычек](../values/strings#unquoted) из двух операндов, разделённых знаком `=`. Это существует для обратной совместимости с очень старым синтаксисом, поддерживаемым только IE.

<div class="grid" markdown>

=== "SCSS"

    ```scss
    .transparent-blue {
      filter: chroma(color=#0000ff);
    }
    ```

=== "SASS"

    ```sass
    .transparent-blue
      filter: chroma(color=#0000ff)
    ```

```css title="CSS"
.transparent-blue {
  filter: chroma(color=#0000ff);
}
```

</div>
