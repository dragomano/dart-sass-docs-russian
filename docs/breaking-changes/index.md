> Новые версии Sass максимально обратно совместимы, но иногда критическое изменение необходимо. Sass должен идти в ногу с развивающейся спецификацией CSS, и старые ошибки в дизайне языка иногда нужно исправлять.

Перед выпуском каждого критического изменения реализации Sass будут выдавать предупреждения об устаревании для таблиц стилей, поведение которых изменится. Когда возможно, эти предупреждения будут включать предложения о том, как обновить устаревшие стили, чтобы сделать их совместимыми с будущими версиями.

Разные реализации имеют разные политики в отношении критических изменений и устареваний. [Dart Sass](/) будет выдавать предупреждения об устаревании как минимум три месяца перед выпуском критического изменения и выпустит критическое изменение с новым номером основной версии, если только это изменение не является необходимым для совместимости с CSS. Изменения совместимости с CSS часто являются как ненарушающими, так и срочными, поэтому они могут быть выпущены с новыми номерами минорных версий.

Эти критические изменения скоро будут выпущены или были недавно выпущены:

* [Устаревшая функция `if()`](../breaking-changes/if-function/) начиная с Dart Sass 1.95.0

* [Приватные переменные в `with`](../breaking-changes/with-private/) начиная с Dart Sass 1.92.0

* [Неправильно размещенные rest-аргументов](../breaking-changes/misplaced-rest/) начиная с Dart Sass 1.91.0

* [Функция `type()`](../breaking-changes/type-function/) начиная с Dart Sass 1.86.0

* [`@import`](../breaking-changes/import/) начиная с Dart Sass 1.80.0

* [Устаревший JS API](../breaking-changes/legacy-js-api/) начиная с Dart Sass 1.79.0.

* [Некоторые способы использования JS API для цвета устарели](../breaking-changes/color-4-api/) начиная с Dart Sass 1.79.0.

* [Ряд функций для работы с цветом устарели](../breaking-changes/color-functions/) начиная с Dart Sass 1.79.0, в пользу новых функций, совместимых с CSS Color 4.

* [Функция `meta.feature-exists()` устарела](../breaking-changes/feature-exists/) начиная с Dart Sass 1.78.0.

* [Смешивание объявлений с вложенными правилами меняет поведение](../breaking-changes/mixed-decls/) начиная с Dart Sass 1.77.7.

* [Функции и миксины, начинающиеся с `--`, устарели](../breaking-changes/css-function-mixin/) начиная с Dart Sass 1.76.0.

* [Передача процентной единицы в глобальную функцию `abs()` устарела](../breaking-changes/abs-percent/) начиная с Dart Sass 1.65.0.

* [Передача `null` в качестве альфа-канала в `new SassColor()` меняет поведение](../breaking-changes/null-alpha) начиная с Dart Sass 1.64.3.

* [Загрузка Sass как экспорта по умолчанию в JS больше не разрешена](../breaking-changes/default-export) начиная с Dart Sass 1.63.0.

* [Переменная может иметь только один флаг `!global` или `!default`](../breaking-changes/duplicate-var-flags) начиная с Dart Sass 1.62.0.

* [Строгие унарные операторы](../breaking-changes/strict-unary/) начиная с Dart Sass 1.55.0.

* [Медиазапросы уровня 4](../breaking-changes/media-logic/) начиная с Dart Sass 1.54.0, и окончание периода устаревания в Dart Sass 1.56.0.

* [Селекторы с недопустимыми комбинаторами являются недопустимыми](../breaking-changes/bogus-combinators) начиная с Dart Sass 1.54.0.

* [`/` меняется с операции деления на разделитель списка](../breaking-changes/slash-div) начиная с Dart Sass 1.33.0.

* [Функции более строги в отношении того, какие единицы измерения они допускают](../breaking-changes/function-units) начиная с Dart Sass 1.32.0.

* [Разбор специального синтаксиса `@-moz-document` будет недопустимым](../breaking-changes/moz-document) начиная с Dart Sass 1.7.2.

* [Составные селекторы не могли быть расширены](../breaking-changes/extend-compound) в Dart Sass 1.0.0 и Ruby Sass 4.0.0.

* [Синтаксис значений пользовательских свойств CSS изменился](../breaking-changes/css-vars) в Dart Sass 1.0.0, LibSass 3.5.0 и Ruby Sass 3.5.0.

## Досрочное включение {#early-opt-in}

Пользователи Dart Sass могут досрочно включить обработку устареваний как ошибок, используя [опцию командной строки `--fatal-deprecation`](../cli/dart-sass/#fatal-deprecation).
