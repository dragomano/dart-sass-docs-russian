> Большая часть дополнительной функциональности Sass представлена в виде новых [at-правил](https://developer.mozilla.org/en-US/docs/Web/CSS/At-rule), которые он добавляет поверх CSS:

- [`@use`](../at-rules/use) загружает миксины, функции и переменные из других таблиц стилей Sass и объединяет CSS из нескольких таблиц стилей вместе.

- [`@forward`](../at-rules/forward) загружает таблицу стилей Sass и делает её миксины, функции и переменные доступными, когда ваша таблица стилей загружается с помощью правила `@use`.

- [`@import`](../at-rules/import) расширяет CSS at-правило для загрузки стилей, миксинов, функций и переменных из других таблиц стилей.

- [`@mixin` и `@include`](../at-rules/mixin) упрощают повторное использование фрагментов стилей.

- [`@function`](../at-rules/function) определяет пользовательские функции, которые можно использовать в [выражениях SassScript](../syntax/structure#expressions).

- [`@extend`](../at-rules/extend) позволяет селекторам наследовать стили друг от друга.

- [`@at-root`](../at-rules/at-root) помещает стили внутри себя в корень CSS-документа.

- [`@error`](../at-rules/error) приводит к сбою компиляции с сообщением об ошибке.

- [`@warn`](../at-rules/warn) выводит предупреждение, не останавливая компиляцию полностью.

- [`@debug`](../at-rules/debug) выводит сообщение для целей отладки.

- Правила управления потоком, такие как [`@if`][], [`@each`][], [`@for`][] и [`@while`][], контролируют, будет ли выведен стиль и сколько раз.

У Sass также есть особое поведение для [обычных CSS at-правил](../at-rules/css): они могут содержать [интерполяцию](../interpolation) и могут быть вложены в правила стилей. Некоторые из них, такие как [`@media`][] и [`@supports`][], также позволяют использовать SassScript непосредственно в самом правиле без интерполяции.
